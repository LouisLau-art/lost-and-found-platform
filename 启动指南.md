# 失物招领平台启动指南

## 新启动脚本使用说明

我已经为您创建了一个功能更完善的启动脚本 `start-project.bat`，它提供了以下功能：

### 功能特点

1. **菜单选择** - 可以选择启动完整平台、仅后端或仅前端
2. **环境检查** - 自动检测Python、Node.js环境和项目依赖
3. **自动修复** - 如果缺少虚拟环境或依赖，会尝试自动创建和安装
4. **用户友好** - 提供清晰的启动信息和提示

### 使用方法

1. **直接运行脚本**：
   - 在文件资源管理器中双击 `start-project.bat`
   - 或者在命令提示符中运行：`start-project.bat`

2. **选择操作**：
   - 输入 `1` - 启动完整平台（前端和后端）
   - 输入 `2` - 仅启动后端服务
   - 输入 `3` - 仅启动前端服务
   - 输入 `4` - 检查环境配置
   - 输入 `0` - 退出脚本

## 服务访问地址

- **后端服务**：http://localhost:8000
- **前端服务**：http://localhost:5173

## 首次运行注意事项

1. **环境要求**：
   - 需要安装Python（推荐3.8+）
   - 需要安装Node.js和npm

2. **首次启动可能需要时间**：
   - 如果虚拟环境不存在，脚本会自动创建并安装依赖
   - 如果前端依赖未安装，脚本会自动执行 `npm install`

3. **服务状态**：
   - 请等待服务完全启动后再访问
   - 后端服务启动成功后会显示 "Uvicorn running on http://localhost:8000"
   - 前端服务启动成功后会显示类似 "Local: http://localhost:5173/"

## 故障排除

### 常见问题

1. **端口冲突**：
   - 如果8000或5173端口已被占用，请关闭占用端口的程序后重试

2. **依赖安装失败**：
   - 确保网络连接正常
   - 可以尝试手动创建虚拟环境和安装依赖

3. **数据库错误**：
   - 如果遇到数据库相关错误，可以尝试运行 `backend\init_db.py` 初始化数据库

## 原始启动脚本

项目中还保留了以下原始启动脚本：

- `start-all.bat` - 启动完整平台
- `start-backend.bat` - 仅启动后端服务
- `start-frontend.bat` - 仅启动前端服务

您可以根据需要选择使用哪个启动脚本。
